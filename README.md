# Практична робота №5

## Завдання варіант №8

Реалізувати некоректний виклик realloc, що втрачає дані при невдалому розширенні буфера.

### Ідея реалізації

Буфер спочатку виділяється через `malloc`. Потім викликається `realloc`, але результат одразу записується в ту саму змінну `buf`. Якщо `realloc` не зможе виділити новий блок пам’яті, він поверне `NULL`, а старий вказівник буде втрачено. У такому випадку програмі вже невідомо де знаходиться попередній блок пам’яті і його неможливо звільнити або використати, а дані фактично втрачаються.

### Висновки

У роботі було продемонстровано некоректне використання функції `realloc`. Якщо результат `realloc` одразу записувати в той самий вказівник, то при помилці виділення пам’яті попередній блок пам’яті буде втрачено, що призводить до витоку пам’яті та втрати даних. Для уникнення цієї проблеми результат `realloc` слід спочатку зберігати у тимчасовій змінній.
